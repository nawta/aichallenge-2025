## ログの書き方

各ログファイルには以下の内容を含めてください：

1. **Overview**: 作業の概要
2. **問題分析**: 解決しようとした課題
3. **実施した変更**: 具体的な変更内容
4. **使用方法**: 新機能の使い方
5. **今後の改善案**: 残タスクや改善点
6. **既知の問題**: 発見した問題点

## ファイル命名規則

```
YYYY-MM-DD_<topic>.md
```

例：
- `2025-12-17_development.md`
- `2025-12-18_bugfix.md`
- `2025-12-20_performance_tuning.md`
